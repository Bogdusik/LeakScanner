# Troubleshooting: Почему уязвимости не исчезают

## Возможные причины:

### 1. Backend не перезапущен
**Самая вероятная причина!** После изменений в коде нужно перезапустить backend:

```bash
cd backend
mvn spring-boot:run
```

Или если используете Docker:
```bash
docker-compose restart backend
```

### 2. Используется старая версия кода
Убедитесь, что:
- Изменения закоммичены и запушены
- Backend пересобран: `mvn clean package`
- Используется правильная версия JAR файла

### 3. Кэш в extension
Очистите кэш extension:
- Откройте `chrome://extensions/`
- Найдите LeakScanner
- Нажмите "Обновить" (Reload)

### 4. Сканер находит реальные hardcoded credentials
Проверьте логи backend (уровень DEBUG):
```bash
tail -f backend/logs/application.log | grep "HARDCODED CREDENTIAL"
```

Это покажет, какие именно файлы и строки находит сканер.

### 5. Изменения не в main ветке
Убедитесь, что изменения объединены в main:
```bash
git checkout main
git log --oneline -5
```

## Как проверить, что изменения применены:

1. Проверьте логи backend при сканировании
2. Убедитесь, что файлы `.md`, `.txt`, `.example` пропускаются
3. Проверьте, что placeholder значения (`your_secure_password_here`) игнорируются

## Если проблема сохраняется:

1. Проверьте логи backend - там будет видно, что именно находит сканер
2. Убедитесь, что backend перезапущен
3. Проверьте, что используется последняя версия кода
